# REITs智能助手 - 定价档位设计方案

## 📋 方案概述

设计基础免费版 + ABC三个档次的收费数据分析模块，实现清晰的产品分级和差异化定价，满足不同用户群体的需求。

---

## 🎯 设计目标

1. **用户体验优先**: 免费版提供足够功能，让用户体验核心价值
2. **清晰的功能分级**: 每个档位有明确的差异化功能
3. **灵活的定价策略**: 支持月付/年付，年付享受折扣
4. **平滑的升级路径**: 用户可以根据需求随时升级
5. **企业级服务**: 为大型企业提供定制化服务

---

## 🔧 技术实现

### 1. 档位服务架构
```
pricing-tier-service.ts
├── PricingTier 枚举
├── PRICING_CONFIG 配置
├── PricingTierService 类
│   ├── getUserTier()
│   ├── getUserFeatures()
│   ├── checkAnalysisLimit()
│   ├── validateRequest()
│   └── getAvailableDataSources()
└── 工具函数
    ├── isHigherTier()
    ├── getTierDisplayName()
    └── getTierPrice()
```

### 2. 权限控制流程
```
用户请求 → 获取用户档位 → 检查功能权限 → 检查使用限制 → 执行分析 → 返回结果
```

### 3. 限制检查机制
- **分析次数**: 每次分析后计数，月末重置
- **数据源权限**: 根据档位配置过滤可用数据源
- **分析半径**: 验证请求半径是否超过档位限制
- **并发用户**: 限制同时在线用户数

### 4. 升级降级流程
```
用户选择新档位 → 计算升级费用 → 支付 → 更新档位 → 立即生效（升级）/月末生效（降级）
```
